# Deck-plan v3 + bounded docking fix — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single-grid split-level interior format with a per-deck 3×3-char tile format that carries per-edge walls/doors/fixtures (spec: `docs/deckplan-format.md`), make server + client collision honor those edges, re-author the Mockingbird as three flat decks, and thread each ship class's own dock-port orientation into mooring (removing a duplicated constant).

**Architecture:** The server `DeckPlan` (Gleam) is the parsed authority; the client mirrors its geometry for local prediction only. Today `DeckPlan.walkable` is one `List(String)` grid whose glyphs (`.`/`#`/`L`/`U`/`2`/`B`) mux two decks together, and walls are implicit (floor-meets-void). v3 makes each deck its own grid of 3×3 tile-blocks: the block **center** says what the tile is (floor / void / `x` stairs), the four **edge-mid** chars say what's on each side (`#` wall / `=` door / letter fixture), corners are cosmetic. Decks connect only via `x` stairs. Collision reads edges. Stations (concourses) share `DeckPlan`, so they migrate too.

**Tech Stack:** Server = Gleam (`server/`, gleam 1.17.0, gleeunit, jesse schemas, postgres for some tests). Client = Godot 4.7 GDScript (`client/`). CI = `.github/workflows/server-test.yml` runs `gleam test` + `gleam format --check`.

### Task 1.2: parse a deck grid from 3×3 blocks

**Files:** Modify `deckplan.gleam`; Test `deckplan_test.gleam`.

**Interface (Produces):** `pub fn parse_deck(name: String, rows: List(String)) -> Result(DeckGrid, String)`

Mapping (per `docs/deckplan-format.md`): a WxH-tile deck is `3H` rows of `3W` chars. For tile `(x,y)` the block origin is `(3x, 3y)`; positions: center `(3y+1, 3x+1)`, N `(3y, 3x+1)`, S `(3y+2, 3x+1)`, W `(3y+1, 3x)`, E `(3y+1, 3x+2)`. Corners ignored. Center: space→Floor, `.`→Void, `x`→Stairs. Edge: space→Open, `#`→Wall, `=`→Door, any other letter→Fixture(that letter). Validate: every row same length, `len%3==0`, row count `%3==0`.

- [ ] **Step 1 (test):**
```gleam
pub fn parse_deck_two_rooms_test() {
  // two 1x1 rooms side by side, double wall between, door in each south edge
  let rows = ["######", "# ## #", "#=##=#"]
  let assert Ok(g) = deckplan.parse_deck("d", rows)
  assert g.width == 2
  assert g.height == 1
  // tile (0,0): floor center, east = Wall, south = Door
  let assert Ok(#(_n, e, s, _w)) = deckplan.edges_at(g, 0, 0)
  assert e == deckplan.Wall
  assert s == deckplan.Door
}
```
- [ ] **Step 2:** `gleam build` → fails (`parse_deck`/`edges_at` missing).
- [ ] **Step 3:** Implement `parse_deck` (string slicing per the mapping) and the trivial `edges_at` accessor (Task 1.3 formalizes queries).
- [ ] **Step 4:** `(cd server && gleam format src test && gleam build)` and run the deckplan tests locally if pg-free (they are): `gleam test` will still try to boot everything — if it hangs on pg, rely on CI. At minimum `gleam build` passes.

### Task 1.3: query helpers

**Files:** Modify `deckplan.gleam`; Test `deckplan_test.gleam`.

**Interfaces (Produces):**
```gleam
pub fn tile_at(g: DeckGrid, x: Int, y: Int) -> Tile          // Void if OOB
pub fn edges_at(g: DeckGrid, x: Int, y: Int) -> Result(#(Edge,Edge,Edge,Edge), Nil)
pub fn is_walkable(g: DeckGrid, x: Int, y: Int) -> Bool      // Floor or Stairs, in bounds
// crossing blocked if EITHER facing edge is Wall/Fixture (spec §Collision)
pub fn edge_blocks(g: DeckGrid, x: Int, y: Int, dir: Dir) -> Bool
pub fn stairs_target(plan: DeckPlan, deck: Int, x: Int, y: Int) -> Result(Int, Nil) // adjacent deck index a Stairs tile connects to
```
where `pub type Dir { N E S W }`. `edge_blocks` checks tile (x,y)'s edge in `dir` OR the neighbor tile's opposite edge (double-wall OR-rule). Door/Open → not blocking; Wall/Fixture → blocking.

- [ ] **Step 1 (test):** open passage vs double wall vs door — one test each, asserting `edge_blocks`.
- [ ] **Step 2–4:** implement, `gleam build`, format.
- [ ] **Step 5:** Commit `feat(deckplan): edge collision + stairs queries`.

**`stairs_target` decision (spec-level):** default rule — a `Stairs` tile at `(x,y)` on deck `d` connects to the same `(x,y)` on the adjacent deck that also has a `Stairs` tile there; if both `d-1` and `d+1` qualify, prefer the one the walker is not already committed to. Read `docs/deckplan-format.md` §Decks; if the Mockingbird layout needs explicit targets, extend the format with an optional per-stairs target and note it in the doc.
