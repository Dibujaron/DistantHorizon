# M3.1 ‚Äî One space, not two (stitched interiors): results

**Date:** 2026-07-17 ¬∑ **Exit criterion met: ship‚Üîconcourse crossing is plain
walking, not a keypress** (proven continuously by
`harness/test_m3_trade.py::test_walk_ashore_and_back_is_just_walking`, and by
that same file's `test_pilot_holds_helm_while_quartermaster_trades` ‚Äî a
quartermaster shanghais onto the pilot's ship by walking, the pilot undocks
and *re-docks* it, exercising the crew-join re-moor path, and the qm still
trades ashore while the pilot holds the helm).

## What M3.1 built

- **`composite` module** ‚Äî a new pure module (`server/src/dh_server/composite.gleam`)
  that builds one combined deck plan from a concourse plan, its authored berth
  tiles, and the ships currently docked there: each ship's tiles/rooms/consoles
  are offset into the concourse frame at its berth (airlock to airlock) and its
  ids namespaced `s{ship_id}:{id}` so several moorings never collide. Two ships
  claiming the same tile is a build error (`"berth_blocked"`); the whole plan is
  normalized to a positive bounding box. This is level generation, not
  coordinate gymnastics ‚Äî the composite is just another `DeckPlan`.
- **Berth moorings + congestion** ‚Äî concourses declare `berths: [[x,y],...]` on
  the world doc (three at Meridian Highport, one at Solis Ring); a docking ship
  claims the lowest free index. A fully-berthed station refuses new dockers
  (`dock_result` reason `"berths_full"`, or `"no_berths"` if the station has no
  berths authored at all) and refuses new *logins* whose spawn ship would need
  a berth that isn't there (`error` code `"station_full"`) ‚Äî an honest, early
  form of M5's anchorage congestion rather than unlimited parking.
- **`space` / `walkers` protocol, replacing `interior`/`concourse`** ‚Äî the
  simulation keeps one `StationSpace` per concourse (composite plan +
  monotonically increasing epoch), rebuilt on every dock/undock/despawn. A
  `space` message pushes a client's current plan plus their own position/seat
  in its frame on login and whenever their space's plan changes; a `walkers`
  message at 15 Hz carries everyone in one space, tagged with that space's id
  and epoch so clients drop frames from a plan they've since left. Flying-ship
  interiors are still spaces too (`ship:N`, epoch always 0, no moorings) ‚Äî the
  same wire shape covers both.
- **Join-on-dock / split-on-undock with crew transfer at the split** ‚Äî
  `character.Place` is `Aboard` (in a *flying* ship's frame) or `OnStation`
  (in a station's composite frame, which now covers standing aboard your own
  *docked* ship). Crew membership (`ship_id`) only changes at the undock split:
  bodies standing on the departing ship's moored tiles leave with it and
  become its crew; bodies on station tiles stay behind. Docking is the
  reverse ‚Äî everyone already aboard is folded into the new composite at their
  ship's mooring, no repositioning needed since the mooring *is* where their tiles
  now live.
- **Clamped interior follow camera** ‚Äî composite plans can outgrow one screen,
  so `client/scripts/interior_view.gd` follows the own character but clamps
  the camera origin so it never scrolls past the plan's edges.
- **X/B crossing deleted** ‚Äî M3's interim "X cycles the airlock" mechanic and
  its docking-collar rendering are gone entirely. Walking down the ramp and
  onto the concourse floor is ordinary `move` input; there is no
  `disembark`/`board` verb left in the protocol or the client's input
  handling.
- **JSON Schemas on the jesse validator** ‚Äî `server/schemas/world.schema.json`
  and `server/schemas/ship_class.schema.json` now validate every world/ship-class
  doc in `worlds/` and `classes/` at test time (`server/test/data_schema_test.gleam`),
  riding the same jesse (Erlang JSON-Schema) FFI wiring the quest system
  introduced. `berths` is schema-checked as an optional array of `[x, y]`
  integer pairs; a malformed one-element berth entry is a dedicated negative
  test.

## Protocol v1 changes

| dir | type | payload | change |
|---|---|---|---|
| ‚Üê | `space` | `space` (`"station:<id>"` \| `"ship:N"`), `epoch`, `plan` (`{grid, walkable, rooms, consoles, spawn_tile}`), `moorings` (`[{ship_id, dx, dy}]`), `you` (`{x, y, seat}`) | **added** ‚Äî the plan a client should now be walking, with their own position/seat in its frame. Sent on login and to every occupant of a space whose plan changed (dock/undock/despawn rebuild). Ship spaces carry epoch `0` and `moorings: []`. Client adopts the plan, snaps to `you`, resets prediction/interpolation. |
| ‚Üê | `walkers` | `tick`, `space`, `epoch`, `characters: [{id, name, x, y, seat}]` | **added** ‚Äî 15 Hz, one message per occupied space, only to that space's occupants. Replaces `interior` and `concourse`. Clients drop walkers whose `space`/`epoch` don't match their current `space` message. |
| ‚Üí / ‚Üê | `board`, `board_result`, `disembark`, `disembark_result` | ‚Äî | **removed** ‚Äî crossing is plain `move` input now; there is no verb. |
| ‚Üê | `interior`, `concourse` | ‚Äî | **removed** ‚Äî superseded by `walkers`. |
| ‚Üê | `dock_result` | `ok`, `reason` | **reasons gained** `"berths_full"` (station has berths, all taken) and `"no_berths"` (station has none authored); `"already_docked"` is now correctly returned when an already-docked pilot at the helm requests dock again (previously fell through to a helm-gate error). |
| ‚Üê | `error` | `code`, `message` | **code gained** `"station_full"` ‚Äî login is refused outright when the spawn station has no free berth for the new player's ship. |
| ‚Äî | seat ids | `"<console_id>"` | **namespaced while docked**: a docked ship's console/room ids appear in the composite as `"s{ship_id}:{console_id}"` (e.g. `"s3:helm_main"`) so several moored ships' consoles never collide. Flying-ship (`ship:N`) spaces keep plain ids. |

Unchanged: `login`, `helm`, `dock`, `undock`, `move`, `sit`, `stand`, `buy`,
`sell`, `get_market`, `get_stats`, `welcome`, `seat_result`, `trade_result`,
`market`, `cargo`, `snapshot`, `stats`.

## World doc schema (`schema: 2`) additions

```jsonc
{
  "stations": [
    {
      "id": "meridian_highport",
      // ‚Ä¶ orbit/dock_radius/crane/concourse/market unchanged from M3 ‚Ä¶
      "berths": [[6, 1], [16, 1], [26, 1]]   // authored mooring anchors, concourse top edge
      // empty or omitted (decode.optional_field, default []) => ships cannot dock here
    }
  ]
}
```

Validated by `server/schemas/world.schema.json`: `berths` is an optional array
of `[x, y]` tile-coordinate pairs, each item constrained to exactly two
integers (`minItems`/`maxItems: 2`); every world doc in `worlds/` and every
ship-class doc in `classes/` is checked against its schema in
`data_schema_test.gleam`, with a dedicated negative test asserting a
one-element berth entry is rejected.

## Known gaps (deliberate, tracked)

- **Composite frame shifts are hard snaps for bystanders on rebuilds** ‚Äî when
  a dock/undock/despawn rebuilds a station's composite, occupants who didn't
  cause the rebuild get a fresh `space` message and their client resets
  prediction/interpolation cold; there's no cross-fade or animated tile shift,
  it's epoch-gated and instant.
- **Berth counts are authored, not derived** ‚Äî a concourse's berth list is
  hand-placed in the world doc; nothing computes how many berths a concourse
  layout could actually fit, so authoring a too-small or overlapping berth
  list is a content bug the schema won't catch.
- **Login refusal instead of an anchorage queue** ‚Äî a full station just
  refuses the login (`error` code `"station_full"`) rather than holding the
  player in a queue or an anchorage orbit; a real queue/anchorage mechanic is
  M5.
- **`berth_blocked` is documented but unreachable for authored layouts** ‚Äî
  `composite.build` returns it when two moorings' tiles collide, but the only
  berth lists that ship today (Meridian, Solis) are spaced so this can't
  happen; it's exercised by `composite_test.gleam` fixtures only.
- **Composite room labels can repeat** ‚Äî each docked ship keeps its own room
  names (e.g. "Helm"), just namespaced by console/room *id*, not by label; a
  concourse with three docked Sparrows shows three rooms all labeled "Helm".
